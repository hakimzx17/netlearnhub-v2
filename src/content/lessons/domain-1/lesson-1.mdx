# OSI Model Foundations

The OSI model is the first mental map that makes the rest of CCNA feel organized. It gives each networking responsibility a layer, which makes packet flow easier to reason about when you troubleshoot.

## Why This Matters

When a packet fails, the OSI model lets you narrow the fault instead of guessing. Link lights and cabling live near the bottom. Addressing, forwarding, and segmentation live higher.

## Encapsulation Walkthrough

1. An application creates data.
2. Transport adds port information.
3. Network adds logical addressing.
4. Data Link adds physical addressing.
5. Physical sends the bits.

## Real-World Lens

When a switch learns MAC addresses or a router chooses a path, both actions make more sense once you can place them on the correct layer.
